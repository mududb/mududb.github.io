# v0.1 发布说明

`v0.1` 是 MuduDB 文档的初始版本，涵盖安装、SQL 使用、过程开发、包部署与架构。

## 新特性

- **SQL CRUD 支持**：`CREATE TABLE`、`INSERT`、`UPDATE`、`DELETE` 以及带谓词与表达式的 `SELECT`。
- **Mudu 过程模型**：使用 Rust（以及实验性的 AssemblyScript）编写存储过程，以 `.mpk` 包部署，并通过 `mcli` 调用。
- **MPK 打包**：将 WebAssembly 组件、DDL、描述符与初始化数据打包为基于 ZIP 的 `.mpk` 归档。
- **多模式服务端后端**：在 `mudud.cfg` 中选择 `Legacy`、`IOUring` 或 `Tokio` 执行模式。
- **每核工作线程模型**：内核管理的工作线程，通过分区本地所有权减少跨核争用。
- **RANGE 分区**：定义全局分区规则、将表绑定到规则，并将分区放置到指定工作线程。
- **格式契约**：page header、tuple binary format、log frame、protocol frame、MPK manifest、server config、file layout、syscall payload 的正式规范。
- **HTTP 管理 API**：通过 `mcli --http-addr` 安装、列出、卸载应用以及查询拓扑。

## 已知限制

- `IOUring` 后端路径上的 SQL 支持尚未实现；当前该后端可使用的方向是 KV 路径。
- 跨多个工作线程的分布式两阶段提交尚未实现。跨 worker 写入会被转发到远程执行，但跨 worker 的完整原子提交尚不支持。
- 不支持二级索引；查询依赖主键与范围扫描。
- `mudup` 发布安装器尚不稳定；开发建议使用源码构建。

## 迁移与兼容性

- v0.1 对所有磁盘格式与网络线格式使用格式版本 `1`。未来版本将提升这些版本号并提供离线迁移工具。
- 配置文件使用 {doc}`Server Configuration Contract <../../reference/contracts/mudud_cfg_v1>` 中描述的 v1 TOML schema。
